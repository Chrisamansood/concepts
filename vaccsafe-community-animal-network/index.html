02 · Google Sheet model

Eight clean tabs—not one giant table

History should be appended as events. Existing animal records should not be overwritten whenever something changes.

Animals
  • AnimalID · Species · Name
  • Sex · Age band · Markings
  • Status · Locality · Visibility
Photos
  • PhotoID · AnimalID
  • DriveURL · ViewAngle
  • UploadedAt · Verified
Updates
  • UpdateID · AnimalID
  • Type · Date · Notes
  • ReporterID · Verification
MedicalEvents
  • EventID · AnimalID
  • Vaccine · Batch · DueDate
  • VetID · CertificateURL
CareCases
  • CaseID · AnimalID · Priority
  • Status · Assignee · Outcome
  • OpenedAt · ClosedAt
Users
  • UserID · Email · Role
  • Locality · Organisation
  • Active · LastSeen
Locations
  • LocationID · Locality
  • LatPublic · LngPublic
  • LatPrivate · LngPrivate
AuditLog
  • ActionID · UserID
  • Action · EntityID
  • Timestamp · Before · After
03 · Delivery roadmap

A practical sequence for QRB

The public app grows only after administrators can trust the underlying records and workflow.

WEEK 01

Data foundation

  • Audit current Sheet
  • Map and clean columns
  • Create stable IDs
  • Define public/private fields
WEEKS 02–03

Core web app

  • Search and profiles
  • QR retrieval
  • Timeline and updates
  • Admin permissions
WEEKS 04–05

Field workflow

  • Fast sighting form
  • Photo and GPS
  • Care case assignment
  • Mobile QA
WEEK 06+

Pilot and learn

  • 100–300 animals
  • 15–25 volunteers
  • Two vet partners
  • Measure care outcomes
04 · Cost and guardrails

Designed to stay near zero

Google quotas are fine for a pilot. Heavy city-wide usage becomes the trigger for a database migration—not an assumption on day one.

Google Sheets database₹0
Apps Script backend + hosting₹0
Custom web interface₹0 runtime

Non-negotiable build checks

Small controls that prevent expensive problems.

✓
Separate public and admin dataNever publish the raw Google Sheet or private coordinates.
✓
Append history, do not erase itEvery medical event and status change remains auditable.
✓
Human-confirm possible duplicatesPhoto matching may suggest; an authorised user decides.
✓
Cache dashboards and watch quotasAvoid re-reading the full Sheet on every interaction.
